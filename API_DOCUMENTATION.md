# API Документация

## Обзор
API предоставляет возможности для работы с сообщениями: получение списка сообщений и создание новых сообщений.

Базовый URL: `http://localhost/api/`

---

## Эндпоинты

### 1. GET /api/index.php - Получение списка сообщений

Возвращает все сообщения из базы данных в формате JSON.

**Метод:** `GET`  
**URL:** `http://localhost/api/index.php`  
**Параметры:** Нет  
**Ответ:** JSON массив объектов сообщений

#### Пример ответа:
```json
[
  {
    "id": 1,
    "text": "Привет, мир!",
    "created_at": "2024-01-15 10:30:00"
  },
  {
    "id": 2,
    "text": "Как дела?",
    "created_at": "2024-01-15 10:35:00"
  }
]
```

#### Пример запроса через curl:
```bash
curl -X GET http://localhost/api/index.php
```

#### Пример в Postman:
1. Метод: `GET`
2. URL: `http://localhost/api/index.php`
3. Headers: не требуются
4. Send

---

### 2. POST /api/index.php - Создание нового сообщения

Создает новое сообщение в базе данных.

**Метод:** `POST`  
**URL:** `http://localhost/api/index.php`  
**Content-Type:** `application/json`  
**Тело запроса:** JSON объект с полем `text`

#### Формат запроса:
```json
{
  "text": "Текст вашего сообщения"
}
```

#### Параметры:
| Параметр | Тип    | Обязательный | Описание                |
|----------|--------|--------------|-------------------------|
| text     | string | Да           | Текст сообщения         |

#### Пример ответа при успехе (201 Created):
```json
{
  "success": true,
  "message": "Сообщение создано",
  "data": {
    "id": 3,
    "text": "Текст вашего сообщения",
    "created_at": "2024-01-15 10:40:00"
  }
}
```

#### Пример ответа при ошибке (400 Bad Request):
```json
{
  "success": false,
  "message": "Поле text обязательно"
}
```

#### Пример запроса через curl:
```bash
curl -X POST http://localhost/api/index.php \
  -H "Content-Type: application/json" \
  -d '{"text":"Привет, это тестовое сообщение!"}'
```

#### Пример в Postman:
1. Метод: `POST`
2. URL: `http://localhost/api/index.php`
3. Headers:
   - Key: `Content-Type`, Value: `application/json`
4. Body → raw → JSON:
```json
{
  "text": "Привет, это тестовое сообщение!"
}
```
5. Send

---

## Коды ответов HTTP

| Код | Описание                          |
|-----|-----------------------------------|
| 200 | Успешное получение данных (GET)   |
| 201 | Успешное создание сообщения (POST)|
| 400 | Ошибка валидации данных           |
| 500 | Внутренняя ошибка сервера         |

---

## Структура данных сообщения

Сообщение представляет собой объект со следующими полями:

| Поле       | Тип    | Описание                     |
|------------|--------|------------------------------|
| id         | integer| Уникальный идентификатор     |
| text       | string | Текст сообщения              |
| created_at | string | Дата и время создания (Y-m-d H:i:s) |

---

## Примечания

1. Все запросы должны отправляться на локальный сервер (localhost)
2. Для POST запросов обязательно указывать заголовок `Content-Type: application/json`
3. Поле `id` и `created_at` генерируются автоматически сервером
4. API поддерживает CORS для запросов с фронтенда
5. Максимальная длина текста сообщения ограничена настройками БД (обычно TEXT тип)

---

## Пример полного цикла работы

1. Получить список существующих сообщений:
```bash
curl -X GET http://localhost/api/index.php
```

2. Создать новое сообщение:
```bash
curl -X POST http://localhost/api/index.php \
  -H "Content-Type: application/json" \
  -d '{"text":"Новое сообщение"}'
```

3. Проверить, что сообщение добавилось:
```bash
curl -X GET http://localhost/api/index.php
```
